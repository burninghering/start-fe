http://주소부분(host):1234(포트)/path/to/resource?(resource path)a=b&x=y(query) GET - 웹서버측 리소스 요청 POST - 내용 전송 PUT - 내용 갱신 위주 DELETE - 파일 삭제 http 메세지 -요청헤더 -응답헤더 캐싱 -응답헤더에 언제까지 문서를 캐시할지, 언제까지 클라이언트에 저장할 지 http의 상태코드 -200 : 성공 -301 : 리다이렉션 -400~403 : 클라이언트 에러 -500 : 서버에러 테스트 도구들 -개발자 도구에서 확인 가능 -Postman : http 리퀘스트를 간단히 테스트 가능 -request bin : 반대로 서버쪽에서 해당 요청들을 테스트 -.http : .http 파일을 만들면 vs코드로 쉽게 확인 가능 -cURL : CLI툴, 터미널 상에서 http의 요청을 테스트 가능 HTTP 스크립팅 -웹 서버와 비동기적으로 데이터를 교환하고 조작하기 위한 XML, JSON -자바스크립트에서 외부 리소스를 가지고 오는 행위 fetch() -다른 공간에서 뭔가를 가지고 올때 "패치"한다 -에러를 캐치할 수 있다 -함수화 해서 사용하면 그다지 길지 않다! JSON -자바스크립트 오브젝트를 표현하는 방식 JSONP -제이썬이 패칭을 해서 외부에 있는 소스를 브라우저에서 가지고 오려면 자바스크립트는 보안때문에 해당 도메인의 소스가 아니면 불러올 수 없는데, JSONP형태로 데이터를 가져올 수 있다
